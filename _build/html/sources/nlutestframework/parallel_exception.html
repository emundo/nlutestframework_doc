async nlutestframework.parallel_exception.run_in_parallel(objects, op, rollback_op, message)[source]¶

Run the same asynchronous operation on multiple objects in parallel. If the operation fails for at least one of the objects, run a rollback operation on the objects that succeeded in the operation. This can be used to implement an “all-or-nothing” mechanism. Either all operations succeed, or the succeeded operations are rolled back.

In case of a failure followed by a rollback, a ParallelException is raised which contains information about the problems that led to the rollback and information about problems during the rollback.

Parameters

  • objects (List[~O]) – The objects to run the operation on.

  • op (Callable[[~O], Awaitable[~R]]) – The operation to run on each object. The object is passed as the first parameter.

  • rollback_op (Optional[Callable[[~O, ~R], Awaitable[Any]]]) – A function which is run once for each object which succeeded in the operation in case at least one operation failed. Receives two parameters: the result of the operation and the object. Pass None to skip the rollback-phase.

  • message (str) – The message to set on the exception that is thrown in case at least one operation failed and the whole set of operations was rolled back.

Return type

List[~R]

Returns

A (future, which resolves to a) list containing the result of the operation applied to each object.

Raises

ParallelException – If at least one operation failed and the whole set of operations was rolled back.
